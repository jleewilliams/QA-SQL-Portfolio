# E-Commerce Website QA Portfolio Project

## Project Overview

This project demonstrates my software quality assurance skills through the creation of test cases and defect reports for a simulated e-commerce web application.

The objective of this project was to apply manual testing principles to common e-commerce workflows, identify potential defects, and document findings using industry-standard QA practices.

The application features tested include:

* User Authentication
* Product Search
* Shopping Cart Functionality
* Checkout Process

# Defect Reports

The following defects were identified and documented:

### BUG_001

Cart total does not update when product quantity changes.

### BUG_002

User can submit checkout without required shipping address.

### BUG_003

Cart total does not recalculate after quantity update.

### BUG_004

User remains authenticated after logout and can access protected pages.

### BUG_005

Relevant search results are not returned for exact product searches.

Total Defects Documented: 5
